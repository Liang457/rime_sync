# rime-server 规划

## 项目概述

rime-server 是一个用 Python 实现的服务器-客户端系统，旨在解决 Rime 输入法配置同步繁琐的问题。项目分为服务器端和客户端（Windows 和 Android），提供配置管理、词库同步、自定义词库生成等功能。

### 核心目标
- 集中管理 Rime 配置（基于 rime-ice 方案），支持多设备同步
- 提供用户自定义词库的生成与集成
- 简化配置文件的修改与分发
- 实现轻量级的客户端 CLI 工具，便于日常同步操作

### 技术栈
- 服务器端：Python + Flask，部署于树莓派 5，监听 10032 端口
- 客户端：Python 脚本（Windows 和 Android）
- 版本控制：Git（用于获取和更新 rime-ice 仓库）
- 数据格式：YAML（Rime 配置）、JSON（API 通信）、tar（批量文件传输）

## 系统架构

```
客户端 (Windows/Android)
        │
        │ HTTP API
        ▼
    rime-server (Flask)
        │
        ├── 配置管理（*.custom.yaml）
        ├── 词库同步（cn_dicts/, en_dicts/, ...）
        ├── 用户输入词库同步（sync/）
        └── 自定义词库生成（makedict/）
```

服务器端维护以下核心目录结构：
- `rime_ice_original/`：从上游 git 克隆的原始 rime-ice 仓库
- `runtime/`：运行时配置副本，实际提供给客户端的文件
- `makedict/`：用户自定义词库生成脚本
- `sync/`：各设备的用户输入词库（userdb 等）
- `logs/`：服务器日志

## 服务器端详细设计

### 1. 基础服务
- 框架：Flask，仅提供 RESTful API，无 HTML 前端
- 端口：10032（可配置）
- 部署：树莓派 5（也可运行于其他 Linux 环境）

### 2. rime-ice 仓库管理
- 使用 git 克隆 rime-ice 仓库到 `rime_ice_original/` 目录（非发行版，直接获取源码仓库）
- 更新时通过比较本地与远程的 commits 判断是否需要获取
- 仅当有新的 commits 时才执行 `git pull`，减少不必要的传输
- 不提交任何更改到原始仓库
- 提供 API 触发更新操作，支持强制更新选项

### 3. 配置文件修改与分发
- 用户可能修改 `rime_ice.dict.yaml` 等文件（如插入多行文本）
- 服务器将 rime-ice 原始文件复制到 `runtime/` 目录，作为实际提供的文件
- 修改操作仅针对 `runtime/` 副本，避免污染原始仓库
- 提供 API 支持行级插入、替换等基础编辑操作

### 4. 用户自定义词库
- 在 `makedict/` 目录下存放用户编写的词库生成脚本（Python）
- 脚本约定：至少接受一个参数（词库版本），生成 `.dict.yaml` 文件
- 示例：`/makedict/yuanshen/main.py` 接收参数 `6.5.1`，生成 `ys.dict.yaml`
- 服务器提供 API 触发脚本执行，并将生成的词库文件移入 `runtime/cn_dicts/`
- 脚本执行环境隔离，避免影响服务器稳定性

### 5. 用户配置管理（*.custom.yaml）
- 主机（master）可以直接上传 `*.custom.yaml` 文件
- 从机（slave）上传需等待主机手动确认（避免配置冲突）
- 提供版本对比和合并建议（预留功能）

### 6. 用户输入词库同步
- 每台设备在 `sync/<device_name>/` 目录下保存其用户输入词库（userdb.txt 等）
- 设备名来自客户端上传的 `installation.yaml` 中定义
- 专用 API（与普通词库分开）：
  - 上传：`/api/sync/upload/tar`（tar 格式）、`/api/sync/upload/file`（单文件）
  - 信息查询：`/api/sync/info` 获取文件列表、大小、SHA3-256 哈希
  - 下载：`/api/sync/get/<设备名>/tar`（tar 包）、`/api/sync/get/<设备名>/file/<文件名>`（单文件）
- 冲突处理：以最新上传为准（可扩展为手动解决）

### 7. 完整同步
- 客户端可请求完整同步，获取除以下文件外的所有配置：
  - `installation.yaml`（设备标识）
  - `.github/`（仓库管理）
  - `build/`（编译产物）
  - `rime_ice.userdb`（用户数据库，由同步机制单独处理）
- 同步包以 tar 格式提供，减少传输次数

### 8. 用户识别
- 个人项目无需复杂认证，但需区分不同设备
- 客户端上传 `installation.yaml` 内容作为设备标识
- 服务器根据标识分配 `sync/<device_name>/` 目录

### 9. API 设计概要

| 端点 | 方法 | 描述 | 请求/响应 |
|------|------|------|-----------|
| `/api/status` | GET | 服务器状态 | `{"version": "1.0", "rime_ice_version": "2024.05.21"}` |
| `/api/rime_ice/update` | POST | 触发 rime-ice 更新（通过 commits 判断） | `{"force": false}` |
| `/api/file/edit` | POST | 编辑配置文件（行插入） | `{"path": "rime_ice.dict.yaml", "line": 17, "content": "新行"}` |
| `/api/makedict/run/<script_name>` | POST | 执行自定义词库脚本（允许网络访问） | `{"version": "6.5.1"}` |
| `/api/config/upload` | POST | 上传 *.custom.yaml | 文件上传（multipart） |
| `/api/sync/upload/tar` | POST | 上传用户输入词库，tar 格式 | 文件上传 + `device` 参数 |
| `/api/sync/upload/file` | POST | 上传用户输入词库，单文件 | 文件上传 + `device` + `filename` 参数 |
| `/api/sync/info` | GET | 获取用户输入词库信息 | 返回文件列表（含哈希、大小等） |
| `/api/sync/get/<设备名>/tar` | GET | 获取用户输入词库的 tar 包 | 返回 tar 文件 |
| `/api/sync/get/<设备名>/file/<file_name>` | GET | 获取用户输入词库的文件 | 返回文件内容 |
| `/api/dict/info` | GET | 获取词库文件信息（cn_dicts 和 en_dicts） | 返回文件列表（含哈希、大小等） |
| `/api/dict/get/tar` | GET | 获取词库的 tar 压缩包 | 返回 tar 文件 |
| `/api/dict/get/file/<file-name>` | GET | 获取词库的单文件 | 返回文件内容 |
| `/api/full_sync/info` | GET | 获取完整配置包文件信息 | 返回文件列表（含哈希、大小等） |
| `/api/full_sync/download` | GET | 下载完整配置包（tar） | 返回 tar 文件 |
| `/api/full_sync/upload` | POST | 上传完整配置包（tar），用于初始化 | 文件上传 + `overwrite` 参数 |
| `/api/device/list` | GET | 列出已注册设备 | `["Win", "n9p", "pad5pro"]` |

（详细 API 文档需单独编写）

### 10. 错误处理
- 统一 JSON 错误格式：`{"error": "错误描述", "code": 400}`
- 常见错误：文件不存在、脚本执行失败、版本冲突
- 日志记录到 `logs/server.log`，支持自动轮转和旧日志归档（`utils/log_manager.py`）
- 归档文件保存在 `logs/archive/`，默认保留 90 天

## 客户端详细设计

### Windows 客户端
- Python 脚本，CLI 界面（`cli.py` + `core/` 模块库）
- 配置文件：`client_config.json`，指定服务器地址、本地 Rime 配置文件夹路径
- 主要功能：
  1. 同步词库（从服务器下载最新词库到本地 `cn_dicts/` 等，支持 lua/opencc 类别，哈希增量）
  2. 同步用户输入词库（上传/下载 `sync/<device>`，SHA3-256 哈希增量对比）
  3. 请求服务器更新词库（可选择更新哪个自定义词库，自动添加词库到 rime_ice.dict.yaml）
  4. 批量脚本执行（`run-all-scripts`）
  5. 请求服务器更新 rime-ice 原始仓库
- 操作流程：调用对应 API，解析响应，更新本地文件

### Android 客户端
- 通过 Termux 运行 Python 脚本实现（无需单独开发 app）
- 配置文件：指定 Rime 配置文件夹路径（通常为 `/sdcard/rime` 或 `~/storage/shared/rime`）
- 特殊文件：`trime.yaml`（Android 端配置文件）需额外处理，与 PC 端的 `weasel.custom.yaml` 类似但配置项不同
- 功能支持：
  - 同步词库（从服务器下载最新词库）
  - 同步用户输入词库（上传/下载 `sync/<device>`）
  - 可选择性地同步配置文件（通过完整同步或单独下载）
- 注意事项：
  - Termux 需要存储权限访问 Rime 配置目录
  - 可通过 Tasker 或类似工具定时触发同步
  - 网络连接可能不稳定，需要重试机制

## 部署指南

### 服务器端（树莓派 5）
1. 安装 Python 3.9+ 和 pip
2. 克隆本仓库
3. 安装依赖：`pip install -r requirements.txt`
4. 配置服务器地址、端口等（编辑 `config/server.json`）
5. 启动 Flask 应用：`python server.py`
6. （可选）设置 systemd 服务，开机自启
   - 复制 `rime-server.service` 到 `/etc/systemd/system/`
   - 注意修改 `WorkingDirectory` 和 `ExecStart` 为实际部署路径

### Windows 客户端
1. 安装 Python 3.9+
2. 下载 `cli.py` 和 `core/` 目录到 Rime 配置目录
3. 编辑 `client_config.json`，设置服务器地址和本地 Rime 目录
4. 通过命令行执行同步操作

### Android 客户端
1. 安装 Termux（从 F-Droid 或 GitHub 获取最新版）
2. 在 Termux 中安装 Python 和相关依赖：
   ```bash
   pkg update
   pkg install python git
   pip install requests
   ```
3. 下载客户端脚本到 Termux：
   ```bash
   cd ~
   git clone <rime-server-client-repo>
   ```
4. 配置 Rime 目录（通常为 `/sdcard/rime`）和服务器地址：
   ```bash
   cd rime-server-client
   cp config.example.json config.json
   # 编辑 config.json，设置服务器 URL 和本地 Rime 路径
   ```
5. 授予 Termux 存储权限：
   ```bash
   termux-setup-storage
   ```
6. 手动运行同步脚本：
   ```bash
   python cli.py
   ```
7. （可选）设置定时同步：使用 Termux:Tasker 插件或 cron 定时执行

## 扩展性与未来工作
- 支持更多 Rime 方案（不仅限于 rime-ice）
- 提供 Web 管理界面，方便可视化操作
- 增加配置冲突自动合并（基于 git merge 策略）
- 支持增量同步，减少数据传输量
- 添加简单的用户认证（可选）
- 客户端自动更新机制

## 注意事项
- 本服务器不涉及敏感数据，但仍需注意文件权限和网络安全
- 自定义词库脚本应受信任，避免执行恶意代码（脚本允许网络访问用于爬虫）
- 定期备份 `runtime/` 和 `sync/` 目录
- 遵循 rime-ice 开源协议（GPL 3.0），不修改其原始文件
