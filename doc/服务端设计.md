# rime-server 设计细节

本文档提供 rime-server 项目的详细技术设计，用于后续开发参考。

## 1. 服务器目录结构

```
rime-server/
├── server.py                 # Flask 主程序
├── config/                   # 配置文件目录
│   ├── server.json          # 服务器基本配置
│   ├── sync.json            # 同步相关配置
│   ├── dict.json            # 词库管理配置
│   ├── script.json          # 脚本管理配置
│   └── devices.json         # 设备信息配置
├── requirements.txt         # Python 依赖
├── rime_ice_original/      # 从上游 git 克隆的原始 rime-ice 仓库
├── runtime/                # 运行时配置副本（实际提供给客户端）
│   ├── cn_dicts/           # 词库文件（包括自定义生成的）
│   ├── en_dicts/
│   ├── lua/
│   ├── opencc/
│   ├── *.yaml              # 方案文件
│   └── ...
├── makedict/               # 用户自定义词库生成脚本
│   ├── yuanshen/
│   │   └── main.py        # 示例脚本
│   └── ...                # 其他脚本
├── sync/                   # 各设备的用户输入词库
│   ├── Win/               # 设备名来自 installation.yaml
│   ├── n9p/
│   ├── pad5pro/
│   └── ...
├── logs/                   # 服务器日志
│   ├── server.log
│   └── archive/            # 旧日志归档目录
├── backups/                # 定期备份（可选）
└── utils/                  # 工具模块
    ├── config_loader.py    # 配置加载器
    ├── error_handler.py    # 错误处理
    └── log_manager.py      # 日志归档管理器
```

## 2. 配置管理

服务器使用多个 JSON 配置文件进行管理，所有配置在服务器启动时一次性加载到内存中，运行时使用内存副本。

### 2.1 server.json - 服务器基本配置
```json
{
  "server": {
    "host": "0.0.0.0",
    "port": 10032,
    "debug": false,
    "log_level": "INFO",
    "log_file": "logs/server.log",
    "log_max_bytes": 10485760,
    "log_backup_count": 5,
    "max_upload_size_mb": 100,
    "allowed_extensions": [".yaml", ".txt", ".dict.yaml", ".tar"]
  },
  "paths": {
    "rime_ice_original": "rime_ice_original/",
    "runtime": "runtime/",
    "makedict": "makedict/",
    "sync": "sync/",
    "backups": "backups/"
  },
  "git": {
    "rime_ice_repo": "https://github.com/iDvel/rime-ice.git",
    "rime_ice_branch": "main",
    "update_check_interval": 3600
  },
  "log_archive": {
    "enabled": true,
    "retention_days": 90
  }
}
```

### 2.2 sync.json - 同步相关配置
```json
{
  "sync": {
    "auto_cleanup_days": 30,
    "max_files_per_device": 100,
    "max_total_size_mb": 1024,
    "conflict_resolution": "latest",
    "hash_algorithm": "sha3-256",
    "manifest_file": "_manifest.json"
  },
  "devices": {
    // 运行时从 sync/ 目录动态生成，不在此配置
  }
}
```

### 2.3 dict.json - 词库管理配置
```json
{
  "dict": {
    "cn_dicts_path": "runtime/cn_dicts/",
    "en_dicts_path": "runtime/en_dicts/",
    "allowed_extensions": [".dict.yaml", ".txt"],
    "auto_update_scripts": {
      "yuanshen": {
        "enabled": true,
        "schedule": "0 0 * * *",
        "version": "auto"
      }
      // 其他脚本配置
    }
  }
}
```

### 2.4 script.json - 脚本管理配置
```json
{
  "scripts": {
    "max_execution_time": 300,
    "max_memory_mb": 512,
    "allow_network_access": true,
    "allowed_hosts": ["*"],
    "trusted_users": ["admin"],
    "log_execution": true
  }
}
```

**脚本信任策略**：
- 脚本由用户手动添加到 `makedict/` 目录，服务器不提供脚本管理 API
- API 只能触发脚本执行，无法添加、删除或修改脚本
- 服务器信任所有已添加的脚本，执行时赋予网络访问权限（用于爬虫）
- 脚本执行环境相对隔离，但信任脚本作者的意图
- `trusted_users` 列表中的设备可以触发脚本执行，其他设备只能读取脚本输出

### 2.5 devices.json - 设备信息配置
```json
{
  "devices": {
    "Win": {
      "type": "master",
      "last_seen": "2026-04-14T12:00:00Z",
      "total_syncs": 15
    },
    "n9p": {
      "type": "slave",
      "last_seen": "2026-04-14T11:30:00Z",
      "total_syncs": 8
    }
    // 其他设备
  }
}
```

### 2.6 配置加载与更新
- **加载时机**：服务器启动时一次性加载所有 JSON 配置文件到内存
- **内存缓存**：配置在内存中缓存，避免频繁磁盘读取
- **配置更新**：支持热重载 API（`POST /api/config/reload`），重新读取配置文件
- **配置验证**：加载时进行 JSON Schema 验证，确保配置格式正确
- **默认值**：缺失的配置项使用合理的默认值

## 3. API 详细设计

### 3.1 通用响应格式

成功响应：
```json
{
  "success": true,
  "data": { ... },
  "message": "操作成功"
}
```

错误响应：
```json
{
  "success": false,
  "error": "错误描述",
  "code": 400
}
```

### 3.2 端点详情

#### GET /api/status
返回服务器状态和 rime-ice 版本。

响应示例：
```json
{
  "success": true,
  "data": {
    "version": "1.0",
    "rime_ice_version": "2024.05.21",
    "uptime": 12345,
    "storage_usage": "2.3GB/16GB"
  }
}
```

#### POST /api/rime_ice/update
触发 rime-ice 仓库更新。服务器将检查远程仓库的最新 commits，如果本地不是最新的，则执行 `git pull` 获取更新。

请求体（可选）：
```json
{
  "force": false
}
```

响应：
```json
{
  "success": true,
  "data": {
    "previous_commit": "abc123",
    "current_commit": "def456",
    "changed_files": ["cn_dicts/base.dict.yaml", ...],
    "upgraded": true,
    "message": "已从 commit abc123 更新到 def456"
  }
}
```

如果已经是最新版本（无需更新）：
```json
{
  "success": true,
  "data": {
    "previous_commit": "def456",
    "current_commit": "def456",
    "changed_files": [],
    "upgraded": false,
    "message": "已经是最新版本"
  }
}
```

#### POST /api/file/edit
编辑配置文件（目前仅支持行插入）。

请求体：
```json

  "path": "rime_ice.dict.yaml",
```

响应：

## 4. 数据结构

### 4.1 设备标识
从 `installation.yaml` 中提取以下字段组合成设备标识：
```yaml
installation_id: "xxx"
device_id: "yyy"
```
服务器将使用 `installation_id` 作为主标识，若不存在则使用文件内容的 SHA3-256 前 8 位。

### 4.2 自定义词库脚本规范
- 脚本必须位于 `makedict/{name}/` 目录下
- 入口文件必须命名为 `main.py`
- 必须接受至少一个命令行参数：词库版本（如 `6.5.1`）
- 可以接受额外的 JSON 参数（通过标准输入或环境变量传递）
- 脚本应在当前目录生成 `.dict.yaml` 文件
- 脚本返回 0 表示成功，非零表示失败
- 标准输出和标准错误将被捕获并记录
- **网络访问**：脚本允许进行网络请求（用于爬虫获取数据生成词库）

**脚本管理原则**：
- 脚本由用户手动添加至 `makedict/` 目录，服务器不提供脚本管理 API
- API 仅能触发脚本执行（`POST /api/makedict/run/{script_name}`），无法修改、删除或上传脚本
- 服务器信任用户添加的脚本，执行时赋予网络访问权限和必要资源
- 脚本执行环境提供基本的资源限制，但基于信任原则不过度限制

示例脚本结构：
```python
#!/usr/bin/env python3
import sys
import json

def main(version, extra_params):
    # 生成词库逻辑
    with open("ys.dict.yaml", "w", encoding="utf-8") as f:
        f.write(f"... 版本 {version} ...")
    return 0

if __name__ == "__main__":
    version = sys.argv[1]
    extra = json.loads(sys.stdin.read()) if not sys.stdin.isatty() else {}
    sys.exit(main(version, extra))
```

### 4.3 同步文件哈希记录
服务器为每个同步文件维护哈希记录：
```json
{
  "luna_pinyin.userdb.txt": {
    "hash": "sha3-256:abc123",
    "modified": "2026-04-14T12:00:00Z",
    "size": 1024
  }
}
```
记录保存在 `sync/{device}/_manifest.json` 中。

## 5. 错误码定义

| 代码 | 含义 | 说明 |
|------|------|------|
| 400 | Bad Request | 请求参数错误 |
| 404 | Not Found | 文件或设备不存在 |
| 409 | Conflict | 版本冲突或文件已存在 |
| 500 | Internal Server Error | 服务器内部错误 |
| 503 | Service Unavailable | 服务暂时不可用（如 git 操作中） |

## 6. 安全考虑

- 文件上传限制：最大文件大小 100MB，tar 文件解包后总大小限制 1GB
- 脚本执行限制：超时 300 秒（5分钟），内存限制 512MB，允许网络访问（用于爬虫生成词库）
- 路径遍历防护：确保用户提供的路径不会逃逸出目标目录
- 日志脱敏：避免记录敏感信息（如文件内容）

## 7. 性能优化

- 使用 LRU 缓存存储常用文件的哈希值
- tar 压缩使用 lzma 模块（Python 内置 `lzma` + `tarfile`）
- 大文件分块传输（预留接口）
- 定期清理旧备份和日志

## 8. 监控与维护

- 健康检查端点：`/api/health`（返回磁盘、内存使用情况）
- **日志轮转**：使用 `RotatingFileHandler`，单文件超过 `log_max_bytes`（默认 10MB）时自动轮转，保留 `log_backup_count`（默认 5）个备份
- **日志归档**：启动时由 `LogManager` 自动将非当天的旧日志压缩归档到 `logs/archive/`
  - 压缩回退链：`7z` → `tar.gz` → 无压缩 `tar`
  - 自动清理超过 `retention_days`（默认 90 天）的归档文件
- 备份策略：每天自动备份 `runtime/` 和 `sync/` 到 `backups/`，保留最近 7 天
