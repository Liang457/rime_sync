# rime-server 客户端设计

本文档详细描述 rime-server 客户端的设计与实现，包括 Windows 和 Android 平台。

## 1. 客户端概述

rime-server 客户端是一个命令行工具，用于与 rime-server 服务器交互，实现 Rime 配置的同步与管理。客户端支持两种运行模式：
- **交互式 CLI 模式**：无参数运行时，提供菜单驱动的交互界面
- **命令行模式**：带参数运行时，直接执行指定操作

## 2. 客户端架构

```
客户端 (Windows/Android)
├── 配置文件管理 (client_config.json)
├── 本地 Rime 配置目录扫描
├── CLI 入口 (cli.py)
│   ├── argparse 22子命令解析
│   └── 交互式菜单
├── 核心逻辑模块 (core/)
│   ├── config.py — 配置管理器
│   ├── api.py — HTTP API 客户端
│   ├── sync.py — 用户词库哈希增量同步
│   ├── dicts.py — 配置增量同步 (SyncState)
│   ├── hash_utils.py — SHA3-256 哈希 + 时间解析
│   ├── fullsync.py — 完整配置同步
│   ├── tar_utils.py — tar 安全解压
│   ├── platform.py — 平台相关操作
│   ├── logs.py — 日志归档与清理
│   └── errors.py — 自定义异常
└── 本地文件操作模块
    ├── tar 打包/解包
    ├── 文件哈希计算 (SHA3-256)
    ├── 目录遍历与过滤
    └── 文件备份与恢复
```

**架构原则**:
- `core/` 模块返回数据/抛异常，不直接 print
- `cli.py` 负责所有 print 输出和异常格式化
- 操作函数使用纯函数签名 `fn(config: ConfigManager, api: APIClient, ...)`
- `ConfigManager` 和 `APIClient` 是类，管理配置和 HTTP 状态

## 3. 配置文件设计

### 3.1 配置文件路径
- Windows: `%APPDATA%\rime-server\client_config.json` 或当前目录 `client_config.json`
- Android (Termux): `~/.config/rime-server/client_config.json` 或当前目录 `client_config.json`

### 3.2 配置文件格式 (client_config.json)
```json
{
  "server": {
    "url": "http://192.168.1.100:10032",
    "timeout": 30,
    "retry_count": 3,
    "verify_ssl": false
  },
  "rime": {
    "config_dir": "C:\\Users\\Username\\AppData\\Roaming\\Rime",
    "platform": "windows",
    "android_trime_config": "/sdcard/rime/trime.yaml"
  },
  "sync": {
    "device_name": "Win",
    "auto_sync_on_start": false,
    "exclude_patterns": [
      "*.userdb.txt.backup",
      "*.log",
      "temp/*"
    ]
  },
  "logging": {
    "level": "INFO",
    "file": "logs/rime_client.log",
    "max_size_mb": 10,
    "archive_enabled": true,
    "archive_retention_days": 90
  }
}
```

### 3.3 配置文件初始化流程
1. 检查配置文件是否存在
2. 如果不存在，创建默认配置并提示用户编辑
3. 验证配置项：
   - 服务器 URL 可达性
   - Rime 配置目录存在性
   - 网络连接测试
4. 自动从 `installation.yaml` 读取 `device_name`（如果未配置）

### 4.1 交互式 CLI 界面

```

 3. 同步用户输入词库（哈希增量）
 4. 同步配置 (cn/en/lua/opencc/全部)
 5. 编辑配置文件
 6. 完整同步 (下载/上传)
 7. 查看同步状态
 8. 设备列表
 9. 健康检查
 10. 修改配置
 11. 退出

选择 [1-11]: 
```

### 4.2 命令行参数模式
支持子命令模式，无参数运行时进入交互式菜单：

```bash
# 无参数：交互式菜单
python cli.py

# 子命令列表
python cli.py status                     # 服务器状态
python cli.py update-rime-ice [--force]  # 更新 rime-ice 仓库
python cli.py run-script <name> <version>  # 执行词库脚本（自动添加词库）
python cli.py run-script <name> <version> --no-add-to-dict  # 不自动添加
python cli.py run-all-scripts <version>  # 批量执行全部脚本
python cli.py list-scripts               # 列出可用脚本
python cli.py edit-file <path> --line <n> --content "<text>"  # 编辑文件
python cli.py upload-config <file>       # 上传配置文件
python cli.py sync-userdb --action upload|download  # 同步用户输入词库（哈希增量）
python cli.py sync-upload-tar <file>     # 上传用户词库 tar
python cli.py sync-download-tar [--since <ts>]  # 下载用户词库 tar
python cli.py sync-dict --category cn|en|lua|opencc  # 同步配置（哈希增量）
python cli.py dict-download-tar          # 下载词库 tar
python cli.py full-sync-download         # 下载完整配置
python cli.py full-sync-upload <file>    # 上传完整配置
python cli.py health                     # 健康检查
```

### 4.3 服务器更新 rime-ice
**API**: `POST /api/rime_ice/update`
**客户端逻辑**:
1. 发送更新请求
2. 轮询状态直到完成（或超时）
3. 显示更新结果（commit 变化、文件变更列表）
4. 建议是否需要重新同步词库

### 4.4 执行自定义词库脚本
**API**: `POST /api/makedict/run/<script_name>`
**客户端逻辑**:
1. 列出可用的脚本（从服务器获取或本地缓存）
2. 提示用户选择脚本和输入版本号
3. 发送执行请求
4. 实时显示脚本输出（如有流式输出支持）
5. 显示执行结果和生成的词库文件信息

### 4.5 同步用户输入词库（哈希增量）
**流程**:
1. **读取设备标识**:
   ```python
   installation_path = os.path.join(config_dir, "installation.yaml")
   installation_id = extract_installation_id(installation_path)
   device_name = config.get("sync.device_name") or installation_id
   ```

2. **哈希增量对比**:
   - 扫描本地 `sync/<device_name>/` 目录，计算所有文件 SHA3-256 哈希
   - 调用 `GET /api/sync/info` 获取服务端文件哈希列表
   - 仅传输哈希不同的文件（增量同步）
   - 冲突按 mtime 较新者胜出

3. **打包上传**:
   - 仅打包有变更的文件为 `.tar`
   - 调用 `POST /api/sync/upload/file` 逐文件上传
   - 或调用 `POST /api/sync/upload/tar` 批量上传

4. **下载其他设备词库**:
   - 调用 `GET /api/sync/info`
   - 对于每个非本设备的词库文件：
     - 对比本地哈希与服务器哈希
     - 仅下载变更文件到 `sync/<其他设备名>/`

### 4.6 同步配置（哈希增量）

**流程**:
1. **获取配置信息**:
   - 调用 `GET /api/full_sync/info?since=<last_sync>` 获取变更文件列表
   - 服务端按文件修改时间过滤，返回变更文件的哈希列表

2. **增量下载**:
   - 通过 `SyncState` (.sync_state.json) 记录本地最近同步时间
   - 调用 `GET /api/full_sync/download?since=<ts>` 仅下载变更文件
   - 支持按类别过滤：cn（中文词库）、en（英文词库）、lua（脚本）、opencc（OpenCC 配置）

3. **备份与替换**:
   - 下载前备份将被替换的文件
   - 直接解压 tar 到配置目录

### 4.7 编辑配置文件
**API**: `POST /api/file/edit`
**客户端逻辑**:
1. 读取本地文件，显示内容
2. 提供编辑界面（行插入、替换、删除）
3. 预览更改
4. 上传到服务器
5. 可选：同时更新本地副本

### 4.8 完整同步
**下载流程** (从服务器获取完整配置):
1. 调用 `GET /api/full_sync/info` 获取文件列表
2. 排除 `installation.yaml`、`.github/`、`build/`、`rime_ice.userdb`
3. 比较本地文件哈希，确定需要下载的文件
4. 调用 `GET /api/full_sync/download` 下载 tar
5. 使用 `tarfile` + `lzma` 解包到临时目录，验证文件完整性
6. 备份当前配置，然后替换

**上传流程** (初始化服务器或完全重置):
1. 收集本地 Rime 配置目录所有文件
2. 排除指定文件/目录
3. 打包为 tar，计算 SHA3-256 哈希
4. 调用 `POST /api/full_sync/upload`
5. 确认操作（因为这会覆盖服务器配置）

### 6.3 Android 特殊考虑
1. **Rime 配置路径**:
   - Trime (Android): `/sdcard/rime/`
   - 需要 Termux 存储权限访问外部存储

2. **配置文件差异**:
   - Android 使用 `trime.yaml` 而非 `weasel.yaml`
   - 同步时需特殊处理平台相关配置

3. **网络限制**:
   - 移动网络可能不稳定，需要重试机制
   - 后台执行可能被系统限制

4. **定时同步**:
   - 使用 Termux:Tasker 插件
   - 或使用 cron 定时任务
   ```bash
    echo "0 2 * * * cd ~/rime-client-android && python cli.py --auto" >> ~/.crontab
   crontab ~/.crontab
   ```

## 7. 错误处理与日志

### 7.1 错误分类
- **网络错误**：连接失败、超时、服务器无响应
- **配置错误**：无效配置、路径不存在、权限不足
- **同步冲突**：文件版本冲突、哈希不匹配
- **API 错误**：服务器返回错误码

### 7.2 恢复策略
1. **网络错误**：指数退避重试，最大 3 次
2. **文件错误**：使用备份恢复，记录错误继续其他操作
3. **冲突解决**：提示用户干预，或使用配置的默认策略

### 7.3 日志记录
客户端使用 `RotatingFileHandler` 实现日志自动轮转，并在启动时自动归档非当天的旧日志。

**日志配置要点**：
- **日志路径**：默认 `logs/rime_client.log`（相对于配置文件所在目录）
- **自动轮转**：单文件超过 `max_size_mb` 时自动轮转，保留最近 5 个备份
- **旧日志归档**：启动时扫描日志目录，将非当天的日志文件压缩归档到 `logs/archive/`
  - 压缩回退链：优先尝试 `7z` → 回退到 `tar.gz` → 最终使用无压缩 `tar`
- **过期清理**：自动删除超过 `archive_retention_days`（默认 90 天）的归档文件

**日志初始化流程**：
```python
from logging.handlers import RotatingFileHandler

log_config = config.get("logging", {})
level = getattr(logging, log_config.get("level", "INFO").upper())
log_file = config_path.parent / log_config.get("file", "logs/rime_client.log")
log_dir = log_file.parent
log_dir.mkdir(parents=True, exist_ok=True)

# 1. 归档旧日志（非当天）
if log_config.get("archive_enabled", True):
    client._archive_old_logs(log_dir, log_config.get("archive_retention_days", 90))

# 2. 配置 RotatingFileHandler
root = logging.getLogger()
root.setLevel(level)
fmt = logging.Formatter("%(asctime)s - %(name)s - %(levelname)s - %(message)s")

fh = RotatingFileHandler(
    str(log_file),
    maxBytes=log_config.get("max_size_mb", 10) * 1024 * 1024,
    backupCount=5,
    encoding="utf-8"
)
fh.setFormatter(fmt)
root.addHandler(fh)

# 3. 控制台输出
ch = logging.StreamHandler()
ch.setFormatter(fmt)
root.addHandler(ch)
```

1. **配置文件安全**：不硬编码敏感信息，使用配置文件

4. **脚本执行**：仅执行信任的服务器返回的脚本
