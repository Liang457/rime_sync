# rime-server 规划大纲

## 1. 项目概述

- **目标**：解决 Rime 输入法配置同步繁琐问题
- **架构**：服务器-客户端（Python + Flask）
- **技术栈**：
  - 服务器：Flask，树莓派 5，端口 10032
  - 客户端：Python CLI（Windows/Android）
  - 版本控制：Git（rime-ice 仓库）
  - 数据格式：YAML、JSON、tar

## 2. 系统架构

```
客户端 → HTTP API → rime-server (Flask)
                    ├── 配置管理 (*.custom.yaml)
                    ├── 词库同步 (cn_dicts/, en_dicts/, ...)
                    ├── 用户输入词库同步 (sync/)
                    └── 自定义词库生成 (makedict/)
```

## 3. 服务器端核心功能

### 3.1 rime-ice 仓库管理

- 克隆 rime-ice 到 `rime_ice_original/`（直接获取源码仓库）
- 通过比较 commits 判断是否需要更新
- 仅当有新的 commits 时才执行 `git pull`
- API 触发更新，支持强制更新选项

### 3.2 配置文件修改与分发

- 原始文件复制到 `runtime/` 作为提供版本
- API 支持行级插入、替换

### 3.3 用户自定义词库

- 脚本存放于 `makedict/`，接收版本参数生成 `.dict.yaml`
- API 触发执行，结果移入 `runtime/cn_dicts/`

### 3.4 用户配置管理 (\*.custom.yaml)

- 主机直接上传，从机需手动确认
- 预留版本对比与合并

### 3.5 用户输入词库同步

- 每台设备对应 `sync/<device_name>/`
- 设备名来自 `installation.yaml`
- 上传/下载（tar 或单文件），带时间戳和哈希
- 冲突处理：最新上传优先

### 3.6 完整同步

- 排除 `installation.yaml`、`.github/`、`build/`、`rime_ice.userdb`
- 提供 tar 包

### 3.7 用户识别

- 无认证，使用 `installation.yaml` 内容作为设备标识

### 3.8 API 概览

| 端点 | 方法 | 描述 |
|------|------|------|
| `/api/status` | GET | 服务器状态 |
| `/api/health` | GET | 健康检查（磁盘、内存） |
| `/api/rime_ice/update` | POST | 触发 rime-ice 更新（通过 commits 判断） |
| `/api/rime_ice/copy_to_runtime` | POST | 复制 rime-ice 到 runtime |
| `/api/file/edit` | POST | 编辑配置文件 |
| `/api/makedict/run/<script>` | POST | 执行自定义词库脚本（允许网络访问） |
| `/api/makedict/list` | GET | 列出可用脚本 |
| `/api/config/upload` | POST | 上传 *.custom.yaml |
| `/api/sync/upload/tar` | POST | 上传用户输入词库，tar |
| `/api/sync/upload/file` | POST | 上传用户输入词库，单文件 |
| `/api/sync/info` | GET | 获取用户输入词库信息 |
| `/api/sync/get/<设备名>/tar` | GET | 获取用户输入词库的 tar 包 |
| `/api/sync/get/<设备名>/file/<文件名>` | GET | 获取用户输入词库的文件 |
| `/api/dict/info` | GET | 获取词库文件的信息 |
| `/api/dict/get/tar` | GET | 获取词库的 tar 包 |
| `/api/dict/get/file/<文件名>` | GET | 获取词库的单文件 |
| `/api/full_sync/info` | GET | 获取完整配置包文件信息 |
| `/api/full_sync/download` | GET | 下载完整配置包（tar） |
| `/api/full_sync/upload` | POST | 上传完整配置包（tar），用于初始化 |
| `/api/device/list` | GET | 列出已注册设备 |
| `/api/config/reload` | POST | 热重载配置 |

### 3.9 错误处理

- 统一 JSON 错误格式
- 日志记录到 `logs/server.log`，支持自动轮转与旧日志归档（`utils/log_manager.py`）
- 归档文件保存在 `logs/archive/`，默认保留 90 天

## 4. 客户端设计

### 4.1 Windows 客户端

- Python CLI，`cli.py` + `core/` 模块库，配置文件 `client_config.json`
- 功能：同步词库（哈希增量，支持 cn/en/lua/opencc）、同步用户输入词库（SHA3-256 diff）、请求更新词库（单个/批量，自动添加词库）、请求更新 rime-ice

### 4.2 Android 客户端

- 通过 Termux 运行 Python 脚本（无需单独开发 app）
- 配置目录：`/sdcard/rime` 或 `~/storage/shared/rime`
- 特殊文件：`trime.yaml`（Android 端配置文件）
- 功能：同步词库、同步用户输入词库、可选择性地同步配置文件
- 注意事项：Termux 存储权限、网络连接稳定性、定时触发

## 5. 部署指南

### 5.1 服务器端（树莓派 5）

1. 安装 Python 3.9+、pip
2. 克隆仓库
3. `pip install -r requirements.txt`
4. 配置并启动 `python server.py`
5. （可选）systemd 服务（参考 `rime-server.service`，注意修改路径）

### 5.2 Windows 客户端

1. 安装 Python 3.9+
2. 下载 `cli.py` 和 `core/` 目录，配置 `client_config.json`
3. 命令行执行同步

### 5.3 Android 客户端

1. 安装 Termux（从 F-Droid 或 GitHub）
2. Termux 内安装 Python、 requests 库
3. 下载客户端脚本，配置 Rime 目录和服务器地址
4. 授予 Termux 存储权限
5. 手动运行同步脚本
6. （可选）设置定时同步（Termux:Tasker 或 cron）

## 6. 扩展性与未来工作

- 支持更多 Rime 方案

- 配置冲突自动合并（git merge）
- 增量同步
- 简单用户认证
- 客户端自动更新

## 7. 注意事项

- 文件权限与网络安全
- 自定义脚本由用户添加，且不能在API端管理，API只能负责触发。服务器应该尽可能信任脚本。
- 定期备份 `runtime/` 和 `sync/`
